# 异常路径与故障恢复分析

## 一、标注图例

```
【★】= 致命异常（程序无法继续，需关机/重启/人工介入）
【▲】= 可恢复异常（程序自动降级或重试后恢复）
【◆】= 警告级别（记录日志，不影响核心功能）
【○】= 正常路径

══════ = 正常执行
- - -  = 异常分支
······ = 超时等待
```

## 三、异常分级与恢复策略汇总

### 致命异常【★】—— 需人工介入

| 异常 | 触发条件 | 系统行为 | 恢复方式 |
|------|---------|---------|---------|
| 加密永久到期 | 到期日 < 今天, 无30天延期 | 弹窗 → 无法进主页 → 设备锁定 | 联系厂商获取新密码 |
| JSON 配置文件损坏 | 文件不存在/格式错误 | 使用默认值（可能功能受限） | 恢复出厂设置(U盘触发) |
| 串口设备不存在 | /dev/ttyS1 未出现 | 所有板卡通信失败 → 自检页全红 | 检查硬件连接/驱动 |
| SD卡满（保存时） | 磁盘空间 = 0 | 配置无法保存 → 下次开机丢配置 | 清理 SD 卡或更换 |
| 关机命令失败 | system("shutdown") 返回非0 | 系统无法自动关机 | 人工断电 |

### 可恢复异常【▲】—— 自动降级或重试

| 异常 | 触发条件 | 降级策略 | 自动恢复 |
|------|---------|---------|---------|
| FPGA模式切换失败 | bFpgaModeAutoSwitchErr==1 | 复位 → 等20秒 → 重试 → 进自检页 | 20秒超时后重试 |
| IPC 模型加载超时 | 15秒内未完成 | 不等了,仅用FPGA传统算法 | IPC 后续自行加载完毕 |
| 气压报警 | 气压 < 阈值 | 停止供料+关阀 | 气压恢复 → 自动恢复分选 |
| 料位报警 | 料斗空 | 停止供料 | 加料后自动恢复 |
| 温度报警 | 温度超限 | 停止供料 | 温度恢复后自动恢复 |
| FPGA 板卡通信超时 | 串口无回包 | 单板标记离线,其他板继续 | 下次通信周期重试 |
| IPC 相机离线 | 网络断开/UDP无回包 | 该相机AI功能不可用,其他相机继续 | 网络恢复后自动重连 |
| 某条指令发送失败 | myQueue push 返回 false | 该条指令丢失,硬件用旧值 | 下次开机重发所有参数 |
| SD卡备份失败 | cp 命令失败 | 仅本地保存,备份跳过 | 下次保存时重试 |
| 启动无配置文件 | startWithoutCnf==1 | 弹窗询问,用户确认后重置 | 用户确认 → 重建默认配置 |

### 警告级别【◆】—— 仅记录日志，不影响功能

| 异常 | 触发条件 | 影响 | 备注 |
|------|---------|------|------|
| breakpad 未启用 | 编译宏未定义 | 崩溃时不生成 dump | 量产版本正常行为 |
| boot.bmp 不存在 | 文件缺失 | 无启动画面 | 纯视觉,不影响功能 |
| MQTT 断连 | 网络问题 | 云平台暂时无法监控设备 | 本地功能完全不受影响 |
| SQLite 写入失败 | SD卡异常 | 统计数据丢失 | 分选功能不受影响 |
| 清理旧日志失败 | 文件系统错误 | 日志文件可能超过30个 | 下次启动再试 |
| 页面创建失败 | CreatePageByID 返回-1 | 无法切换到该页 | 记录日志,手动重试 |
| IPC 关机确认超时 | IPC 响应慢 | 不等了,继续关机 | IPC 侧自行关机 |
| 日志文件读取失败 | 文件损坏 | 跨天归档跳过 | 下次启动再试 |

## 五、降级路径图

```
                   分选功能完整性

  全功能（最优）:  FPGA + IPC + MQTT + 所有板卡正常
        │
        ├── IPC 离线
        │     └──→ 降级: FPGA 传统算法（无AI分类,精度下降,速度不变）
        │
        ├── 某块相机板离线
        │     └──→ 降级: 该视不参与分选（其他视正常）
        │
        ├── 某块阀板离线
        │     └──→ 降级: 该通道不喷阀（物料直接落入好料斗,有漏检风险）
        │
        ├── 控制板离线
        │     └──→ 降级: 无法供料/清灰（分选功能无法运行）
        │           → 停留在自检页
        │
        ├── 接口板离线
        │     └──→ 降级: 无法通信所有相机和阀
        │           → 分选功能完全无法运行
        │           → 停留在自检页
        │
        ├── FPGA 模式切换失败
        │     └──→ 降级: 复位FPGA → 等20秒 → 重试
        │           → 仍失败 → 进自检页
        │
        ├── MQTT 断连
        │     └──→ 降级: 云平台功能不可用（远程控制/OTA/数据上报）
        │           → 本地分选功能完全不受影响
        │
        ├── SD 卡满
        │     └──→ 降级: 配置无法保存、统计数据丢失
        │           → 分选功能继续进行（当前参数在内存中有效）
        │           → 下次开机恢复旧配置
        │
        └── 加密到期
              └──→ 降级: 设备锁定（分选功能无法运行）
                    → 仅显示加密对话框
```

## 六、关键设计特点总结

1. **分选功能不依赖云**：MQTT/SQLite/日志都是附属功能，断网/断存储不影响核心分选
2. **逐板降级**：一块板卡离线不影响其他板卡，局部失效局部降级
3. **自检页是安全港**：任何板卡通信失败都不会崩溃，而是停在自检页让工程师判断
4. **加密是单点瓶颈**：加密到期直接锁死设备——这是商业策略而非技术限制
5. **保存失败不阻止关机**：日志 error 后继续关机流程，不掉入"保存失败→无法关机"的死循环
6. **IPC 加载超时不阻塞**：15 秒超时后跳过，保留 FPGA 兜底
