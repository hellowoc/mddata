# 提示与调试方法汇总

色选机工程中所有输出、提示、弹窗、日志的方式。按"是否给用户看"分两大类。

---

## 一、给用户看的（界面提示）

### 1. 底部提示条 — g_infoWidget()

**类型**：`myInfoWidget`，全局单例 `g_infoWidget()`。定义在 `mysortwidget/mymessagebox.h:35`。

**外观**：屏幕底部弹出一行文字，灰底黑字，几秒后自动消失。

| 方法 | 作用 |
|------|------|
| `setLabelText("文字")` | 设置提示文字 |
| `delayShow()` | **立即渲染**然后显示（内部用 processEvents 强制绘制） |
| `delayHide()` | 隐藏 |
| `showSecs(int secs=2)` | 显示指定秒数后自动隐藏 |

**典型用法**：

```cpp
// 短暂提示，自动消失
g_infoWidget().setLabelText("网络连接成功");
g_infoWidget().showSecs();           // 默认 2 秒

// 需要立刻显示出来，然后做耗时操作
g_infoWidget().setLabelText("网络测试中...");
g_infoWidget().delayShow();          // 立刻渲染
// ... 执行 ping 等耗时操作 ...
g_infoWidget().delayHide();
```

**适用场景**：**不需要用户确认**的提示。操作成功了/失败了/进行中，告知一声即可。

---

### 2. 标准确认框 — myMessageBox

**类型**：`myMessageBox`，定义在 `mysortwidget/mymessagebox.h:15`。

**外观**：居中弹出的模态对话框，带图标 + 文字 + 按钮。

**创建**：

```cpp
myMessageBox msgBox(类型, "提示文字");
int ret = msgBox.exec();             // 阻塞等待用户点击
if (ret == QDialog::Accepted) {      // 点了"确定"
    // ...
}
```

**四种类型**（`mymessagebox.h:10-13`）：

| 宏 | 值 | 按钮 | 用途 |
|------|----|------|------|
| `MSG_QUES` | 0 | 确定 + 取消 | 询问"是否继续？" |
| `MSG_INFO` | 1 | 仅确定 | 信息通知 |
| `MSG_WARN` | 2 | 确定 + 取消 | 警告"可能有风险" |
| `MSG_ABOUT` | 3 | 仅确定 | 关于信息 |

**适用场景**：**需要用户明确确认**的操作。改 IP 前确认、关机前确认等。

---

### 3. 长文本确认框 — myLongMessageBox

**类型**：`myLongMessageBox`，定义在 `mysortwidget/mywid.h:130`。

**外观**：和 `myMessageBox` 一样，但能显示长文本（如 VPN 证书内容）。

**典型用法**（IPSetWidget 申请证书时）：

```cpp
QFile *myfile = new QFile("/sdcard/cfg/zk.ovpn");
myfile->open(QIODevice::ReadOnly | QIODevice::Text);
QString str = myfile->readAll();              // 读整个文件内容

myLongMessageBox msgBox(MSG_QUES, str);       // 显示完整证书内容
int ret = msgBox.exec();
if (ret == QDialog::Accepted) {
    // 用户确认使用此证书
}
```

**适用场景**：需要**用户查看并确认**的长文本内容（证书、协议、日志）。

---

## 二、不給用户看的（开发调试和日志）

### 4. Qt 调试输出 — qDebug()

**头文件**：`<QDebug>`

**输出位置**：终端（stdout）。只有在 Debug 模式下才输出，Release 版本可以全局关闭。

```cpp
qDebug() << "ping Fail!";
qDebug() << "view:" << view << "unit:" << unit;
```

**特点**：链式 `<<` 拼接，自动加换行，不需要 `\n`。

**适用场景**：**开发调试**——临时加的打印，发布前可以关掉或删除。

---

### 5. C 标准调试输出 — printf()

**头文件**：`<stdio.h>`

**输出位置**：终端（stdout）。与 qDebug 的区别——不能自动关闭，永远输出。

```cpp
printf("OK: Obtain Socket Despcritor sucessfully.\n");
printf("\n222 recv[%d][%d] cmd[%02x]\n", view, unit, nCmd);
```

**适用场景**：底层 C 风格的代码（`mynetwork.cpp`、`myfastnetwork.cpp` 中大量使用），以及受 `struGsh.debugMode` 控制的调试输出。

---

### 6. Log4Qt 日志系统 — logger()

**头文件**：`<logger.h>`，`LOG4QT_DECLARE_QCLASS_LOGGER` 宏声明

**输出位置**：日志文件（持久化到磁盘）。

**级别**（从低到高）：

| 方法 | 用途 | 例子 |
|------|------|------|
| `logger()->info()` | 一般信息 | `logger()->info("save json begin")` |
| `logger()->warn()` | 警告 | `logger()->warn("imagepath is not standard")` |
| `logger()->error()` | 错误 | `logger()->error("requestToken timeout expire")` |
| `logger()->fatal()` | 致命错误 | （本工程未使用） |

**类的声明方式**：在类定义中加入 `LOG4QT_DECLARE_QCLASS_LOGGER` 宏，之后所有成员函数内都可以直接用 `logger()`。

**适用场景**：**持久化日志**——需要事后排查问题的关键事件。关机记录、串口命令、MQTT 消息、文件保存、异常警告等。

---

### 7. 自定义错误日志 — g_SelfLog

**类型**：自定义日志类，全局变量 `g_SelfLog`。

**输出位置**：独立的错误日志文件。

```cpp
g_SelfLog->writeErrorLog("send udp packet id 255 error !!!");
g_SelfLog->writeWarnLog("...");
```

**适用场景**：需要**单独记录**的错误事件，与 log4qt 的日志分开管理。

---

## 对比速查表

| 方法 | 给用户看 | 阻塞 | 持久化 | 适用场景 |
|------|---------|------|--------|---------|
| `g_infoWidget()` | 是 | 否 | 否 | 操作反馈："成功"/"失败"/"进行中" |
| `myMessageBox` | 是 | **是** | 否 | 需要用户确认："是否继续？" |
| `myLongMessageBox` | 是 | **是** | 否 | 长文本确认（证书、协议） |
| `qDebug()` | 否 | 否 | 否 | 开发调试，临时打印 |
| `printf()` | 否 | 否 | 否 | 底层 C 代码输出，debugMode 控制 |
| `logger()` | 否 | 否 | **是**（写文件） | 持久化日志，事后排查 |
| `g_SelfLog` | 否 | 否 | **是**（写文件） | 独立错误日志 |

---

## 本工程中的实际使用场景

```
操作成功/失败         → g_infoWidget().setLabelText("xxx") + showSecs()
操作进行中 + 等结果   → delayShow() + 干活 + delayHide()/showSecs()
危险操作确认          → myMessageBox(MSG_QUES, "是否xxx?") + exec()
长文本展示            → myLongMessageBox(MSG_QUES, 证书内容) + exec()
开发时跟踪变量        → qDebug() << "变量:" << value
底层 socket 调试      → printf("OK: xxx\n")
关键事件持久化        → logger()->info("xxx")  串联口命令、MQTT、配置保存
独立错误记录          → g_SelfLog->writeErrorLog("xxx")
```

## 选择逻辑

```
需要用户看见吗？
  ├── 不需要 → 需要持久化？
  │             ├── 是 → logger() 或 g_SelfLog
  │             └── 否 → qDebug() 或 printf()
  │
  └── 需要 → 需要用户确认吗？
              ├── 要确认 → 文字很长？
              │             ├── 是 → myLongMessageBox
              │             └── 否 → myMessageBox
              │
              └── 不需要确认 → g_infoWidget()
```
