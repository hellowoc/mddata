# 全局参数

色选机工程中所有全局共享的数据结构。定义在 `CameraTest(aiuse)/zksort/global/globalparams.h`。

---

## struGsh (struShare)

**定义位置**：`globalparams.h:1361`

**声明**：`globalparams.h:1847`
```cpp
extern struct struShare struGsh;
```

**作用**：运行时全局共享状态。存储当前正在发生的一切——运行时间、报警状态、分选状态、当前页面、温湿度传感器值等。

**读写者**：所有线程都能读写。串口线程写入传感器值，主线程定时器读取并刷新界面。

**关键字段**（与网络/通信相关）：

| 字段 | 类型 | 含义 |
|------|------|------|
| `nCounter` | int | 本次运行时间（秒） |
| `bSortStart` | int | 分选是否启动（0=停止，1=运行） |
| `nPageSm` | int | 当前显示的页面 ID |
| `nLayer` | int | 当前选中的层 |
| `nView` | int | 当前选中的视野 |
| `nUnit` | int | 当前选中的单元 |
| `sRowRed/Green/Blue/Inf[1024]` | int | 当前行 RGB+红外波形数据 |
| `debugMode` | bool | 调试模式开关（控制 printf 输出） |
| `nRemoteImageCaptureIsRun` | int | 远程抓图是否运行中 |

## struCnfg (struCnfGlobal)

**定义位置**：`globalparams.h:1656`

**声明**：`globalparams.h:1850`
```cpp
extern struct struCnfGlobal struCnfg;
```

**作用**：全局配置参数。存储用户保存的配置——层数、视野数、每视野的相机数量、IP 地址、报警阈值等。程序启动时从 JSON 配置文件加载。

**关键字段**（与网络/通信相关）：

| 字段 | 类型 | 含义 |
|------|------|------|
| `nLayerTotal` | int | 总层数 |
| `struLayerInfo[n].nViewChainTotal` | int | 第 n 层的视野链总数 |
| `struLayerInfo[n].nViewIsUsed[j]` | int | 第 n 层第 j 视野是否启用 |
| `struLayerInfo[n].nViewBoardType[j]` | int | 视野对应的板卡类型（映射到 board_h） |
| `struLayerInfo[n].stuViewInfo[j].nViewUnitId[k]` | int | 单元 ID |
| `struIpcCameraParams[n][j][k]` | 结构体 | IPC 相机参数（IP 地址、类型、ROI） |
| `nDPMachine` | int | 是否 DP 机型（决定是否显示 IPC 报警） |
| `nAlarmValid[ALARM_IPC]` | int | IPC 报警是否启用 |

---

## struIpcShare (struIpcShareInfo)

**定义位置**：`globalparams.h:202`

**声明**：`globalparams.h:1848`
```cpp
extern struct struIpcShareInfo struIpcShare;
```

**作用**：IPC AI 工控机的运行时状态。存储每台 IPC 的版本、模型加载状态、CPU/内存使用率、温度、丢行数等。

**关键字段**：

| 字段 | 类型 | 含义 |
|------|------|------|
| `struIpcInfo[view][unit]` | stu_ipc_info | IPC 相机信息（版本、模型状态、CPU、内存、温度等） |
| `struIpcStatus[view][unit]` | stu_ipc_status | IPC 状态（坏料计数） |
| `struIpcModelAble[view][unit]` | stu_ipc_model_able | 可用模型列表 |
| `struIpcModelClass[view][unit]` | stu_ipc_model_class | 模型类别信息 |
| `isIpcRestart[view][unit]` | int | IPC 是否自动重启（切换模型后需重启） |
| `allIpcConnecOk` | int | 所有 IPC 连接是否正常 |

**数据流**：SendThread 收到 IPC 心跳回包 → `processTheDatagram()` 写入 `struIpcShare` → UI 定时器读取刷新。

---

## struCnfp (struCnfProfile)

**定义位置**：`globalparams.h:1121`

**声明**：`globalparams.h:1852`
```cpp
extern struct struCnfProfile struCnfp;
```

**作用**：当前方案的参数——算法灵敏度、颜色设置、IPC ROI 等。与 struCnfg 的区别：struCnfg 是机型配置（固定），struCnfp 是方案参数（可切换方案）。

---

## 结构体命名规律

| 前缀 | 含义 | 示例 |
|------|------|------|
| `struCnfg` | 配置（Configuration） | `struCnfg.nLayerTotal` |
| `struGsh` | 全局共享（Global Share） | `struGsh.bSortStart` |
| `struCnfp` | 配置方案（Config Profile） | `struCnfp.nIpcRoiHeight[view]` |
| `struIpc` | IPC 相关 | `struIpcShare.struIpcInfo[][]` |
