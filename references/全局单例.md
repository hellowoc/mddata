# 全局单例

色选机工程中全局唯一的对象。单例（Singleton）意味着整个程序只有一个实例，通过全局变量访问。

---

## g_Udp (MyUdp)

**声明**：`CameraTest(aiuse)/zksort/bus/myudpthread.h:354`
```cpp
extern MyUdp g_Udp;
```

**定义**：`CameraTest(aiuse)/zksort/bus/myudpthread.cpp:10`
```cpp
MyUdp g_Udp;
```

**类型**：`MyUdp`，继承自 `QObject`。定义在 `myudpthread.h:265`。

**作用**：IPC 命令通信的核心对象。管理 UDP 命令的组装、发送、回包解析。

**关键方法**：

| 方法 | 作用 |
|------|------|
| `initSocket()` | 创建 SendThread 对象 |
| `pushUdpCmdData(nCmd, needReturn, viewId, unitId, dataLen, data)` | 组装 12 字节协议头 + CRC，调用 addPacket 发送 |
| `processTheDatagram(QByteArray, quint8)` | 收到回包后按命令字分发解析 |
| `Stop()` | 停止 SendThread |

**所属线程**：`MyUdp` 对象本身在主线程创建。它管理的 `SendThread` 在独立线程运行。

**初始化位置**：`globalflow.cpp:10044`
```cpp
g_Udp.initSocket();
```

## myFlow (GlobalFlow)

**声明**：`CameraTest(aiuse)/zksort/global/globalflow.h:369`
```cpp
extern GlobalFlow myFlow;
```

**类型**：`GlobalFlow`，继承自 `QObject`。定义在 `globalflow.h:31`。

**作用**：全局流程控制器。整个工程的"中枢神经"，几乎所有跨模块的操作都通过 `myFlow` 调用。

**关键方法**（网络相关）：

| 方法 | 作用 |
|------|------|
| `getIpcInfo()` | 查询 IPC 状态 |
| `loadIpcModel()` | 加载 IPC AI 模型 |
| `configIpcCamera()` | 配置 IPC 相机参数 |
| `configIpcSortStart()` | 启动 IPC 分选 |
| `configIpcSortStop()` | 停止 IPC 分选 |
| `shutdownIpc()` | 关闭/重启 IPC |
| `msleep(int ms)` | 让当前线程休眠 ms 毫秒 |

**所属线程**：主线程。

**初始化位置**：`globalflow.cpp`（在 `initAll()` 函数中初始化）。

## 访问模式

所有全局单例都可以在工程的**任意位置**直接使用，无需传参：

```cpp
// 在某个 widget 的槽函数中直接调用
g_Udp.pushUdpCmdData(CMD_UDP_IPC_REQ_INFO, true, viewId, unitId, 0, NULL);
myFlow.getIpcInfo();
g_MainManager().SetBottomEnable(true);
```
