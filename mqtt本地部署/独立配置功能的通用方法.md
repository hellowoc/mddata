# 独立配置功能的通用方法

## 诊断框架：四层关卡模型

一个已写好但被禁用的功能，必在以下四层中至少有一层阻断。按顺序排查：

```
编译阶段                       链接阶段              运行阶段
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ 第一层：源码 │ → │ 第二层：初始化│ → │ 第三层：链接 │ → │ 第四层：运行时 │
│ 预处理关卡   │    │ 调用关卡     │    │ 库关卡       │    │ 配置+库部署   │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
```

---

## 第一层：源码关卡 — 编译阻断点

### 三种阻断模式

| 模式 | 语法 | 常见位置 | 说明 |
|------|------|---------|------|
| 完全禁断 | `#if 0 ... #endif` | `.h` 类定义、`.cpp` 全局变量 | 左右两边 `#endif` 必须配对 |
| 平台分割 | `#ifdef Q_OS_UNIX` | `.cpp` 实现代码 | 同一功能可能多个文件被分块 |
| 逐行注释 | `// func()` / `// new Xxx` | 单条调用语句 | 创建、启动、注册 |

### 排查命令

```bash
# 1. 扫全工程 #if 0 块
grep -rn '^#if 0$' zksort/ --include='*.h' --include='*.cpp' | grep -v build/

# 2. 扫平台分割点
grep -rn '#ifdef Q_OS_UNIX' zksort/ --include='*.cpp' | grep -v build/

# 3. 扫注释掉的函数调用
grep -rn '//.*new\|//.*start()' zksort/global/ --include='*.cpp'
```

### #if 0 嵌套删除规则

删 `#if 0` 时必须同时删对应的 `#endif`。判断配对标的方法：从 `#if 0` 往下数 `#if`/`#ifdef` 嵌套深度，找到同深度的 `#endif`。

```c
#if 0              // ← 删这行
#ifdef Q_OS_UNIX
...代码...
#endif
#endif             // ← 同时删这行
```

## 第二层：链接关卡 — 库的状态

### 判断流程

```bash
# 1. 看 .pro 里 LIBS 行是否被注释
grep -n '#LIBS\|#.*-l' zksort.pro

# 2. 看对应 3rdparty 目录有没有 .so
ls -la 3rdparty/库名/lib/unix/

# 3. 看 .so 架构是否匹配
file 3rdparty/库名/lib/unix/*.so

# 4. 系统有无替代品
dpkg -l | grep 库名-dev
```

### SOURCES vs LIBS 不匹配陷阱

源码在 SOURCES 但库在 LIBS 被注释 = 链接时报 undefined reference。这两个必须同步启/禁。

## 第三层：初始化关卡 — 功能创建/启动

功能对应的线程/服务对象是否被创建和启动。集中检查点：`globalflow.cpp` 的 `initAll()` 函数。

```bash
grep -rn 'new.*Thread\|new.*Srv\|->start()' zksort/global/globalflow.cpp | grep -v '^.*//'
```

## 第四层：运行时关卡 — 配置与 .so 部署

### 配置文件

程序从固定路径读配置，需从源码逆推：

```bash
grep -rn 'QSettings\|/sdcard/\|./lang/' zksort/global/ --include='*.cpp'
```

常见配置路径：
- `/sdcard/cfg/CFG_APPSet.ini` — 全局设置
- `./lang/*.ini` — 语言文件（相对工作目录）
- `/app/root.crt` — TLS 证书

### .so 运行时加载

编译时链接 `-l` ≠ 运行时能找到。启动脚本 `LD_LIBRARY_PATH` 必须包含所有库目录。

```bash
# 检查运行时依赖
ldd dist/zksort | grep "not found"
```

## 执行流程

### 诊断阶段（只查不改，找全所有控制点）

```
步骤1 → 扫 #if 0:   grep -rn '#if 0' zksort/ | grep -v build/
步骤2 → 扫 .pro:    grep -n '#.*-l\|LIBS' zksort.pro
步骤3 → 扫 初始化:  grep -rn 'new.*Thread\|//.*new' zksort/global/
步骤4 → 扫 库文件:  for d in 3rdparty/*/; do echo "$d: $(find $d -name '*.so' | wc -l)"; done
步骤5 → 扫 运行时:  grep -rn '/sdcard/\|/app/' zksort/ --include='*.cpp' | grep -v build/
步骤6 → 汇总:       列出所有阻断点，画依赖关系图
```

### 执行阶段（按依赖顺序改）

```
库文件就位 → 解封源码 → 恢复初始化 → 改 .pro → 配运行时
```

每一步验证：
```bash
# 改源码 → 编译
qmake zksort.pro && make -j$(nproc)
# 编译通过 → 检查符号
nm dist/zksort | grep 功能关键词
# 运行测试
LD_LIBRARY_PATH=... timeout 5 ./dist/zksort
```

## 故障速查表

| 症状 | 最可能原因 | 查哪 |
|------|------|------|
| `fatal error: xxx.h` | 头文件路径不对 | .pro 的 INCLUDEPATH |
| `undefined reference` | .pro LIBS 被注释 | .pro + 3rdparty/ |
| `'XxxClass' was not declared` | 类在 #if 0 里 | .h 文件 |
| `#endif without #if` | 删 #if 0 忘关配对 | 嵌套层级 |
| `libxxx.so: cannot open`（运行时） | LD_LIBRARY_PATH 没包库 | 启动脚本 |
| `skipping incompatible`（链接时）| .so 架构不匹配 | file 命令查架构 |
| 编译通过但功能不生效 | 初始化被 // 注释 | globalflow.cpp |
| `No such file or directory`（运行时） | 配置文件缺失 | /sdcard/ 路径 |
| `error while loading shared libs` | .so 不在 LD_LIBRARY_PATH | ldd 检查 |
