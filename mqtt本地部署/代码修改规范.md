# 代码修改规范

## 原则

**Git 负责记录历史，注释负责解释原因。**

- 旧代码不保留在文件里（`git log -p` 能找回来）
- 不做 `#if 0` / `#if 1` 新旧代码堆叠
- 只在新代码处加简短注释说明为何改动

## 格式

```cpp
// [zcy 2026.06.15] mqtt config
// 为了适配 MQTT 的第三方支持和各项文件依赖所进行的改动
```

### 规则

- 文件顶部加一行 `// [zcy YYYY.MM.DD] 主题`
- 同一主题的所有文件用同一个主题名（如 `mqtt config`）
- 第二行写改动原因说明

### 只有关键改动需要额外行注释

普通改动（取消注释、取消 `#if 0`）一行标记就够了。逻辑有变化时才在改动的代码行上方加一句解释：

```cpp
// [zcy 2026.06.15] mqtt config
#ifdef Q_OS_UNIX
    // 原代码被 #if 0 屏蔽，解封后恢复 MQTT 线程创建
    myMqttThread = new mqttThread;
#endif
```

## 示例

| 场景 | 做法 |
|------|------|
| `#if 0` 解封 | 直接删 `#if 0`，关闭对应的 `#endif`，文件顶加标记 |
| 取消注释 | 直接删 `//`，文件顶加标记 |
| 新增代码 | 文件顶加标记，关键逻辑加行注释 |
| 修改逻辑 | 文件顶加标记，修改处加行注释说明变更 |

## 配合 Git

```bash
# 每次改动集中提交，message 格式：主题: 描述
git commit -m "mqtt config: 解封 MQTT 类定义和线程创建"
git commit -m "mqtt config: 取消注释链接选项，添加 OpenSSL 依赖"

# 查看改动历史
git log --oneline            # 提交列表
git log -p                   # 每次提交的完整 diff
git show <commit_hash>       # 某次提交的详情
git blame <file>             # 每行代码是谁改的
```
