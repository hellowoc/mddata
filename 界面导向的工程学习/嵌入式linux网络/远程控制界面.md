## 1. 页面结构

RemoteWidget 是一个**QTabWidget**，包含两个标签页：

```
远程控制页 (RemoteWidget)
├── Tab 0："网络配置" (IPSetWidget)
│   ├── eth0 网卡 IP/掩码/网关
│   ├── eth1 网卡 IP/掩码/网关（如果没有 eth1 则隐藏）
│   ├── "修改" 勾选框 → 解锁编辑
│   ├── "应用" 按钮 → 保存到 CFG_APPSet.ini
│   ├── "网络测试" 按钮 → ping 外网
│   ├── "连接" / "断开" 按钮 → OpenVPN 连接云端
│   ├── "VPN证书申请" / "导入证书" 按钮
│   ├── "切换" 按钮 → 交换 eth0/eth1 的 IP
│   └── pingThread（子线程）：每 5 秒 ping VPN 服务器
│
└── Tab 1："PLC控制" (AutoCtrlBoardIPSetWidget)
    ├── 自动控制板 IP/端口/网关/子网掩码
    ├── "查询" 按钮 → 串口查询控制板当前配置
    ├── "应用" 按钮 → 串口下发新配置到控制板
    └── Modbus IP + 波特率设置
```

**Tab 1 的可见性**：仅当 `struCnfg.bautoplcctrlEnable == 1` 时启用（`remotewidget.cpp:22`）。

## 2. RemoteWidget 容器

**代码**：`remotewidget.cpp`（仅 32 行）

```cpp
void RemoteWidget::showPage(bool bshow)
{
    if (bshow) {
        basewidget *widget = (basewidget*)ui->tabWidget->currentWidget();
        widget->showPage();                                            // 激活当前标签页
        ui->tabWidget->setTabEnabled(1, struCnfg.bautoplcctrlEnable == 1);  // 控制 PLC 标签是否可用
    }
}
```

RemoteWidget 没有任何业务逻辑——它只是一个**壳**，把 QTabWidget 包了一层，负责标签切换时调用子页面的 `showPage()`。

## 3. Tab 0：网络配置（IPSetWidget）[[ipsetwidget.c|源码解读]]

### 3.1 页面生命周期

```
构造函数
  ├── m_bModify = false（编辑锁，默认锁定）
  ├── initUi() — 设置输入框类型（IP 格式的 textType / 端口格式的 intType）
  ├── new pingThread — 创建 ping 线程
  └── connect(pingThread, pingFailSig, on_btn_disconnect_clicked)
        // ping 失败 → 自动断开 VPN 连接

showPage(true)
  ├── updateUi() — 根据 m_bModify 启用/禁用输入框
  ├── 从 CFG_APPSet.ini 读取 eth0/eth1 的 IP/掩码/网关 → 填入输入框
  ├── 如果 eth1 没有配置 IP → 隐藏 eth1 相关的所有控件
  └── 如果运行模式是工程师模式 → 隐藏大部分按钮（只读查看）
```

### 3.2 双网卡 IP 配置

**控件**：
- `IP_LineEdit` / `Mask_LineEdit` / `Gateway_LineEdit` — eth0 的 IP/掩码/网关
- `IP_LineEdit_2` / `Mask_LineEdit_2` / `Gateway_LineEdit_2` — eth1 的 IP/掩码/网关

**数据来源**：`QSettings(CFG_APPSet, IniFormat)` → `[eth0]` / `[eth1]` 节。

**保存**：`on_okBtn_clicked()`（第 91-118 行）

```cpp
void IPSetWidget::on_okBtn_clicked()
{
    if (m_bModify) {
        QSettings setting(CFG_APPSet, QSettings::IniFormat);
        setting.beginGroup("eth0");
        setting.setValue("ipStr",     ui->IP_LineEdit->text());
        setting.setValue("maskStr",   ui->Mask_LineEdit->text());
        setting.setValue("gatewayStr", ui->Gateway_LineEdit->text());
        setting.endGroup();
        // eth1 同理...
        setting.sync();               // 立即写入磁盘

        myFlow.getSetting();          // 通知全局重新加载配置
        m_bModify = false;            // 重新锁定编辑
        ui->checkBox->setChecked(false);
        updateUi();
    }
}
```

**编辑锁机制**：默认所有 IP 输入框是**灰色不可编辑**的。需要先勾选"修改"复选框（`on_checkBox_clicked`），设置 `m_bModify = true`，输入框才变为可编辑。这是防止误操作的保护措施。

### 3.3 切换网口 IP 按钮

`on_changeBtn_clicked()`（第 442-483 行）：交换 eth0 和 eth1 的 IP 配置。弹确认框 → 交换两个输入框的文本 → 写入 INI 文件 → `myFlow.getSetting()` 重新加载。

### 3.4 网络测试按钮

`on_btn_nettest_clicked()`（第 162-189 行）：执行 `ping -w 5 ntp.ntsc.ac.cn`（5 秒超时），检查输出中是否包含 "ttl"（Linux）/ "TTL"（Windows）来判断通断。显示"网络连接成功"或"网络连接失败"。

### 3.5 VPN 连接 / 断开

**连接**：`on_btn_connect_clicked()`（第 193-229 行）

```
1. 检查 tun0 虚拟网卡是否已存在（getLocalVirtualAddr）
   ├── 不存在 → 执行 system("cd /app/;./openvpn --config /sdcard/cfg/zk.ovpn &")
   │             启动 OpenVPN 后台进程
   │   └── connetToServer()：轮询 30 次（每次等 1 秒）
   │       检查 tun0 网卡是否出现
   │         → 成功：禁用连接按钮，启动 pingThread
   │         → 失败：killall openvpn，提示连接失败
   └── 已存在 → 直接显示 VPN IP
```

**断开**：`on_btn_disconnect_clicked()`（第 273-287 行）

```
system("killall openvpn") → 杀掉 OpenVPN 进程
停止 pingThread → 恢复按钮状态
```

**pingThread**（第 291-366 行）：一个独立的 QThread，`run()` 中 `while(m_bIsRunning)` 循环，每 20 秒 ping 一次 VPN 网关（`10.18.0.1`）。如果 ping 失败 → `emit pingFailSig()` → 自动调用 `on_btn_disconnect_clicked()` 断开 VPN。

**线程**：pingThread 在**子线程**中执行 `QProcess`（ping 命令），不会阻塞 UI。但 `connectToServer()` 的 30 秒轮询在**主线程**中执行（`myFlow.sleep(1)` × 30），连接期间 UI 会卡。

### 3.6 VPN 证书

- **申请**：`on_btn_request_cert_clicked()` → HTTP 请求云端获取 VPN 证书 → 保存到 `/sdcard/cfg/zk.ovpn`
- **导入**：`on_btn_import_cert_clicked()` → 从 U 盘复制 `zk.ovpn` 到 `/sdcard/cfg/`

### 3.7 工程师模式隐藏

`showPage()` 第 59-67 行：当 `struGsh.bRunMode == RUNMODE_ENGINEER` 时，所有操作按钮（连接、断开、导入证书、网络测试、VPN申请、应用、修改复选框、IP显示标签、切换）全部隐藏。工程师模式下只能**查看**，不能**修改**。

## 4. Tab 1：PLC 控制板设置（AutoCtrlBoardIPSetWidget）

### 4.1 页面生命周期

```
构造函数
  ├── m_nCtrlBoardPort = struCnfg.nAutoBoardPort    // 控制板端口
  ├── m_nModbusIP = struCnfg.nModbusIP              // Modbus 从站地址

```

### 4.2 控件

| 控件 | 创建 | 功能 |
|------|------|------|
| `IP_LineEdit` | .ui | 自动控制板 IP |
| `Port_LineEdit` | .ui | 控制板端口（1-65535） |
| `Gateway_LineEdit` | .ui | 网关 |
| `SubnetMask_LineEdit` | .ui | 子网掩码 |
| `IpqueryBtn` | .ui | **查询**：通过串口查询控制板当前配置 |

### 4.3 查询按钮

`on_IpqueryBtn_clicked()`（第 107-139 行）：

```cpp
// 1. 心跳锁（防止查询期间心跳干扰）
struCnfg.bUpdateStatus = 1;

// 2. 串口发送查询命令
char data[SEND_PACKET_RECTRL_BOARD_SIZE];
memset(data, 0, SEND_PACKET_RECTRL_BOARD_SIZE);
mySerial.pushCom2CmdData(CMD_CTRL_BOARD_IP_PORT_QUERY, 0, BOARD_TYPE_CONTROL, 0x0e, data, 3);
//                        ↑ 命令字：查询控制板IP端口

// 3. 等 3 秒（等控制板回复）
myFlow.sleep(3);

// 4. 刷新显示（updateSlt 已被串口回包信号触发，struCnfg 已更新）
updateUi();

// 5. 解开心跳锁
struCnfg.bUpdateStatus = 0;
```

**背后线程**：命令由 `mySerial.pushCom2CmdData` 发送（在主线程入队，由串口发送线程发出）。回包由串口接收线程解析 → 写入 `struCnfg.nAutoBoardIpAddr[]` 等字段 → `emit signalCtrlBoardIpPortQuery()` → 主线程 `updateSlt()` → `updateUi()` 刷新显示。

### 4.4 应用按钮

`on_IpSetBtn_clicked()`（第 141-193 行）：

```cpp
// 1. 解析界面输入
IPList = ui->IP_LineEdit->text().split(".");
struCnfg.nAutoBoardIpAddr[0..3] = IPList[0..3].toInt();
struCnfg.nAutoBoardPort = m_nCtrlBoardPort;
// 网关、子网掩码同理...

// 2. 组装串口命令
data[0..3]  = nAutoBoardIpAddr[0..3];  // IP 地址 4 字节
data[4..5]  = port/256, port%256;      // 端口 2 字节
data[6..9]  = gateway[0..3];           // 网关 4 字节
data[10..13] = subnetMask[0..3];       // 子网掩码 4 字节

// 3. 发送
mySerial.pushCom2CmdData(CMD_CTRL_BOARD_IP_PORT_SET, 0, BOARD_TYPE_CONTROL, 0x0e, data, 3);
```

### 4.5 Modbus 设置

`on_ModbusSetBtn_clicked()`（第 195-215 行）：通过串口发送 `CMD_PLC_MODBUS_SET` 命令，携带 Modbus 从站地址和波特率索引。

## 5. 完整数据结构

```
RemoteWidget (QTabWidget)
│
├── Tab 0: IPSetWidget
│   数据源: QSettings(CFG_APPSet.ini) [eth0]/[eth1]
│   线程: pingThread (QThread) — 子线程，每20秒ping VPN网关
│   命令: system() 调用 openvpn/killall/ping
│
└── Tab 1: AutoCtrlBoardIPSetWidget
    数据源: struCnfg.nAutoBoardIpAddr[] / nAutoBoardPort / nGateway / nSubnetMask
    命令: mySerial.pushCom2CmdData() — 通过串口发送/接收
    线程: 主线程发命令，串口线程收发，信号回主线程刷新
```

## 6. MQTT 远程锁屏页（RemoteControlWidget）

之前讲的那个锁屏弹窗（`mainpage/remoteControl.cpp`）确实存在，但它**不出现在 SysStateWidget 的左侧栏**中。它是在云端通过 MQTT 下发远程控制请求时，由 `mqttMsgParaseThread` 主动调用 `g_MainManager().ShowWidget(SM_REMOTE_CONTROL_WIDGET)` 弹出的。你在正常操作中看不到它，除非云端有人请求远程操控你的机器。
