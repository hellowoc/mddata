## 1. 这个页面在哪

`MachineInfoWidget` 是系统信息页 QTabWidget 的一个标签页。`SysInfoWidget`（`sysinfowidget.cpp`）创建 QTabWidget → 把 `MachineInfoWidget` 作为 tab 加入。

---

## 2. 页面生命周期

```
SysInfoWidget 构造
  → new MachineInfoWidget
    → 构造函数：setupUi、创建 myTimer、连接 timeout → TimeOutSlt()
    
用户切换到系统信息页
  → showPage(true)
    → myTimer->start(1000)     启动 1 秒定时器
    → updateInfo()             立即刷新一次

定时器每 1 秒触发
  → TimeOutSlt()
    → updateInfo()             从全局变量读取最新数据，刷新所有标签

用户切换到其他页面
  → showPage(false)
    → myTimer->stop()          停止定时器，不再刷新
```

**线程**：全部在主线程（GUI 线程）。`updateInfo()` 只读全局变量，没有 I/O 操作。

### 3.1 本次运行时间 — thisRunLabel

**创建**：`machineinfowidget.ui` → `ui->thisRunLabel`

**刷新函数**：`updateInfo()` 第 63-64 行

```cpp
str = QString("%1 : %2:%3:%4")
    .arg(g_myLan().ids_thisRunTime)                    // 多语言文本："本次运行时间"
    .arg(struGsh.nCounter / 3600)                      // 小时
    .arg((struGsh.nCounter % 3600) / 60)               // 分钟
```

**线程跨越**：`myUpdateStatusThread`（子线程）写 `struGsh.nCounter` → 主线程定时器读。

## 5. 关键设计点

**1. 为什么每次刷新都读 QSettings 而不是缓存 IP？**  
因为 IP 可能在 WiFi 配置页被修改，配置文件会更新。每次读文件保证显示的是最新值。代价是每秒一次文件 I/O，但 INI 文件极小，完全可以接受。

**2. 为什么报警控件要动态 show/hide？**  
因为这是**通用固件**，适配不同机型。有的机型没有气压传感器，有的没有 IPC。`struCnfg.nAlarmValid[]` 数组控制哪些报警是"这个机型有的"，界面动态适配。

**3. 所有数据刷新都在主线程中完成。**  
全局变量由子线程写入，主线程只读。读写时间差最大 1 秒，对状态显示来说可以接受。没有使用互斥锁。
