## 第 1 章出现的

### QTabWidget（§1）

Qt 的标签页容器。在本工程中，`SysInfoWidget` 用 `QTabWidget` 包含三个标签页：操作日志、网络信息、更多信息。`MachineInfoWidget` 是其中一个标签页。

### basewidget（§1）

本工程自定义的基类控件（`mysortwidget/basewidget.h`）。所有页面控件都继承它。提供统一的接口：`showPage(bool)`（页面切换时调用）、`retranslateUiWidget()`（多语言切换时调用）。

### setupUi（§1）

Qt Designer 生成的函数。把 `.ui` 文件中定义的控件创建出来并设置属性。在构造函数中调用 `ui->setupUi(this)` 后，`ui->xxx` 的控件指针才可用。

## 第 2 章出现的

### showPage(bool bshow)（§2）

`basewidget` 的虚函数。当用户切换到此页面时，框架调用 `showPage(true)`；切换离开时调用 `showPage(false)`。

本页的 `showPage(true)` 启动定时器并立即刷新；`showPage(false)` 停止定时器。

**为什么需要显式启停**：Qt 的 `QWidget::show()/hide()` 只是控制可见性，定时器还在跑。`showPage` 机制确保不可见的页面不消耗 CPU。

### QTimer::start(1000)（§2）

启动定时器，参数 1000 = 1000 毫秒 = 1 秒。之后每 1 秒发射一次 `timeout()` 信号，触发 `TimeOutSlt()` 槽。

### myTimer->stop()（§2）

停止定时器。不再发射 `timeout()` 信号。

## 第 3.1 节出现的

### g_myLan()（§3.1）

全局多语言对象。返回当前语言下的翻译文本。

```cpp
g_myLan().ids_thisRunTime  // 中文环境 → "本次运行时间"
                            // 英文环境 → "This Run Time"
```

所有界面文本都通过 `g_myLan()` 获取，支持中英文切换。`ids_` 前缀表示这是一个字符串 ID。

### QString::arg()（§3.1）

`QString` 的格式化方法。`%1`、`%2`、`%3` 等占位符依次被 `arg()` 的参数替换：

```cpp
QString("%1:%2:%3").arg(10).arg(30).arg(5)  → "10:30:5"
```

### nCounter / 3600（§3.1）

整数除法。`nCounter` 是秒数，除以 3600（一小时的秒数）得到小时数。

### nCounter % 3600（§3.1）

取余运算。`% 3600` 得到不满一小时的剩余秒数，再除以 60 得到分钟。

## 第 3.3 节出现的

### QSettings（§3.3）

见 [[references/Qt网络类#QSettings]]。

### CFG_APPSet（§3.3）

宏定义（`constant.h`），展开为配置文件路径字符串，如 `"/config/app_config.ini"`。

### beginGroup / endGroup（§3.3）

`QSettings` 的方法。`beginGroup("eth0")` 进入 INI 文件的 `[eth0]` 节，之后 `value("ipStr")` 读取的是该节下的键。`endGroup()` 退出当前节。

### QSettings::IniFormat（§3.3）

指定使用 Windows INI 格式（`[section]\nkey=value`）。

### 从磁盘读 INI 文件（§3.3）

`QSettings::value()` 返回的是**从磁盘文件读取的缓存值**。每次调用 `value()` 不一定真正读磁盘——Qt 会缓存。但如果外部程序修改了 INI 文件，`QSettings` 可能不会立刻感知。

### bSortStart（§3.5）

## 第 3.6 节出现的

### if(0)（§3.6）

C/C++ 中 `if(0)` 永远为假。这是一种**注释掉代码**的方式——代码还在，但不被执行。与 `/* */` 注释不同，`if(0)` 内的代码仍然受编译器检查，不会因为注释嵌套导致语法错误。

## 第 3.9 节出现的

### #ifdef Q_OS_UNIX（§3.9）

C/C++ 预处理指令。`#ifdef Q_OS_UNIX` 和 `#endif` 之间的代码只在 Linux/Unix 平台编译。Qt 根据目标平台自动定义 `Q_OS_UNIX` 宏。

本工程中 MQTT 和 WiFi 相关功能都是 Linux 专属的，Windows 编译时跳过。

### bFlagMqttConnect（§3.9）

`struGsh` 的成员。MQTT 线程在连接成功时设为 `true`，断开时设为 `false`。

## 第 3.12 节出现的

### 动态数组（§3.12）

```cpp
QLabel *tempSensorLabel[CTRL_ALARM_MAX_TEMP_SENSOR];
```

这是一个**指针数组**——5 个 QLabel 指针。在构造函数中从 UI 文件逐个赋值：

```cpp
tempSensorLabel[0] = ui->ctrlboardTempSensorValue;
tempSensorLabel[1] = ui->tempSensorValue1;
// ...
```

这样可以用 `for` 循环统一操作多个标签，而不是写 5 段重复代码。

### QString::number(x, 'f', 1)（§3.12）

格式化浮点数为字符串。`'f'` = 常规格式，`1` = 保留 1 位小数。

```cpp
QString::number(36.789, 'f', 1) → "36.8"
```

## 第 4 章出现的

### myUpdateStatusThread（§4）

全局状态轮询线程。在 `globalflow.cpp:10041-10042` 创建并启动：

```cpp
myUpdateStatusThread = new updateStatusThread;
myUpdateStatusThread->start();
```

这个线程负责：
1. 每秒 `struGsh.nCounter++`
2. 定时发送串口命令查询报警状态
3. 发射 `sUpdateIpcInfo()` 信号通知 IPC 信息页刷新

### 无锁共享（§4）

多个线程读写同一个全局变量而不加锁。条件是：
- 只有一个线程写入
- 读取方可以接受稍旧的值（不需要精确到微秒的一致性）

本工程中，子线程写 `struGsh`，主线程定时器读，满足以上条件。

## 第 5 章出现的

### QSettings::value() 性能（§5）

`QSettings::value()` 虽然是文件 I/O，但 Qt 内部使用了缓存——第一次读取时加载整个 INI 文件到内存，后续 `value()` 直接从内存读取，不实际访问磁盘。所以每秒调用一次没有性能问题。
