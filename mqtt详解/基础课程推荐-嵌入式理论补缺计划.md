# 基础课程推荐 — 嵌入式理论补缺计划

## P0：立刻补，直接影响你读懂本工程代码

### 1. 计算机网络（重点：TCP/IP 协议栈）

**暴露的问题**：不知道应用层/传输层/网络层的区别、不知道 TCP 为什么有三次握手、不知道 socket 到底是什么。

**需要补的具体知识点**：

| 知识点 | 在本工程中的对应 |
|--------|---------------|
| TCP 三次握手/四次挥手 | MQTT 连接 Broker 时的底层行为 |
| UDP vs TCP 的区别 | 为什么三通道全用 UDP 而非 TCP |
| IP 地址/端口/子网掩码 | eth0/eth1 的 IP 配置、bind() 的参数含义 |
| socket API（socket/bind/connect/send/recv/select） | mynetwork.cpp 的 BSD Socket 代码 |
| 网络分层（五层/四层模型） | 理解 MQTT→TCP→IP→以太网的封装链 |

**B 站搜**：`计算机网络 谢希仁` 或 `计算机网络 韩立刚`，只看前 6 章（物理层→传输层），应用层可以跳过。韩立刚的版本配套有抓包实验，对本工程特别有帮助。

**时间**：2 周，每天 1-2 小时。只看到传输层 TCP/UDP 即可，应用层 HTTP/DNS 等暂时不需要。

---

### 2. Linux 系统编程（重点：文件 I/O + 进程线程 + 网络 socket）

**暴露的问题**：不清楚系统调用是什么、不知道 write/read/send/recv 是用户态到内核态的跨越、不知道为什么多线程要分离。

**需要补的具体知识点**：

| 知识点 | 在本工程中的对应 |
|--------|---------------|
| 文件描述符（fd） | socket() 返回的 int 到底是什么 |
| write/read 系统调用 | sendto/recvfrom 的底层实现 |
| 进程 vs 线程 | QThread 的 run() 和 Linux 原生 pthread 的关系 |
| 阻塞 vs 非阻塞 I/O | select() 为什么能做到"不阻塞检查" |
| 互斥锁/条件变量 | QMutex/QMutexLocker 的底层原理 |

**B 站搜**：`Linux系统编程 黑马程序员` 或 `Linux C语言编程 系统编程`。重点看文件 I/O、进程、线程、网络 socket 四个章节。

**时间**：3 周，每天 1-2 小时。关键是**边看边写代码**——每个知识点都在 Linux 上用 C 写一遍，验证理解。

## P1：一个月内补，帮你理解"为什么代码这样写"

### 3. 操作系统（重点：进程/线程 + 调度 + 同步互斥 + I/O 事件驱动）

**暴露的问题**：不理解内核态/用户态的边界、不知道为什么 select 比 Qt 信号快、不理解 TCP 重传是内核做的不是应用做的。近期集中暴露：不理解 Qt 信号槽跨线程排队机制的本质（事件循环 + 事件队列）、QWaitCondition/QMutex 的底层原理、`exec()` 为什么能让一个线程同时处理收包+定时器+信号槽。

**B 站搜**：`操作系统 王道考研` 或 `操作系统 清华大学 向勇`。

**重点章节**（按照当前需求的优先级排列）：

| 知识点 | 和你当前问题的直接关系 |
|--------|----------------------|
| 进程 vs 线程（进程管理章节） | QThread 和 Linux 原生 pthread 的关系 |
| CPU 调度算法（调度章节） | 为什么多线程能"同时"跑、sleep/msleep 挂起时发生了什么 |
| 信号量与互斥锁（同步与互斥章节） | QMutex、QWaitCondition 的底层 syscall 实现 |
| 中断与事件驱动（I/O 系统章节） | select()、Qt 事件循环、readyRead 信号的本质 |

**学完后验证标准**：回到 `myudpthread.cpp` 的 `SendThread::run()`，能解释为什么 `exec()` 一调，线程就能同时处理 UDP 收包 + 定时器 + 信号槽，而不需要一个 `while(1)` 死循环。

**跳过**：死锁（虽有用但非最紧迫）、页面置换算法、磁盘调度——这些跟嵌入式 Linux 应用开发关系不大。

**时间**：4 周。

---

### 4. 计算机组成原理（重点：CPU 中断 + 总线 + 存储层次）

**暴露的问题**：不太明显——因为本工程是应用层开发，硬件细节被 Linux 内核屏蔽了。但做嵌入式不可能永远待在应用层。

**B 站搜**：`计算机组成原理 王道考研` 或 `计算机组成原理 哈工大 刘宏伟`。

**只看这几章**：
- CPU 结构与指令执行（理解代码怎么变成 CPU 指令）
- 中断与异常（理解串口/网卡数据到达时 CPU 怎么知道的）
- 存储器层次（寄存器→Cache→内存→硬盘，理解为什么内存比硬盘快 1000 倍）
- 总线（理解 CPU 和网卡/串口怎么通信）

**时间**：4 周。不需要全部看完，挑上面四章即可。

## P2：有余力时补，提升代码质量和工程能力

### 5. C++ 深入（重点：内存管理 + 多态 + 模板 + STL）

目前能读懂本工程的 C++ 代码，但缺乏对 C++ 语言机制的深层理解。本工程大量使用 Qt 的封装，掩盖了 C++ 底层细节。

**B 站搜**：`C++ 侯捷` 或 `C++ Primer 视频`。侯捷的视频（STL 源码分析、C++11 新特性）对理解本工程中的 QList/QMap 和 `override`/`nullptr` 有帮助。

## 推荐的执行计划

```
第 1-2 周：计算机网络（TCP/IP + socket）
第 3-5 周：Linux 系统编程（文件I/O + 进程线程 + 网络socket）
第 6-9 周：操作系统（进程管理 + 内存管理 + I/O子系统）
第 10-13 周：计算机组成原理（CPU + 中断 + 总线 + 存储层次）
第 14+ 周：C++ 深入（侯捷视频 + STL 源码）
```

每周每天 1-2 小时课外时间，到第 5 周结束时，应该能**自己写出一个 TCP 客户端/服务端的完整代码**——这是检验"网络 + Linux 系统编程"两块是否融会贯通的硬标准。

## 一个重要提醒

这些课程都是 CS 科班大二/大三的内容。光学工程硕士的数学和逻辑基础够——缺的不是智力，是**概念词汇表和底层机制的直觉**。科班生学这些时有实验室环境（写 OS、写协议栈），当前只有阅读理解代码的环境。所以**每学一个概念就回到本工程代码中找对应**——比如学了 socket() 后重新读 `mynetwork.cpp:udpBind()`，以前看不懂的代码会变得透明。
