## 一、源码修改

### 1. `3rdparty/qextserialport/qextserialport.pro`（第 18-19 行）

**修改**：添加 `INCLUDEPATH += $$MOC_DIR`

```diff
 MOC_DIR = build/moc
 OBJECTS_DIR = build/obj
-
+INCLUDEPATH += $$MOC_DIR
+
 include($$PWD/src/qextserialport.pri)
```

**原因**：qextserialport 的 `.cpp` 文件末尾有 `#include "moc_xxx.cpp"`，moc 生成的文件在 `build/moc/` 目录下，而 Qt Creator 编译时 include 路径不包含此目录，导致 `fatal error: moc_qextserialport.cpp: No such file or directory`。终端 `qmake + make` 不受影响。

## 四、Qt Creator 配置

### 7. `solution.pro.user`（工程配置文件）

关键配置项：

| 配置 | 值 | 说明 |
|------|-----|------|
| BuildDirectory | `/home/jie/桌面/aiuse-qtrun` | **禁用影子构建**，直接在源码目录编译 |
| Kit | `PC`（Id=6152...）| x86_64 GCC + 系统 Qt5.15（`/usr/lib/qt5/bin/qmake`）|
| Executable | `zksort/dist/qt_run.sh` | 通过脚本启动，确保工作目录正确 |
| LD_LIBRARY_PATH | 四个库路径 | 运行时加载 3rdparty 库 |
| UseTerminal | `true` | **关键：独立终端运行**，否则文字不渲染 |

### 8. Qt Creator Kit 配置（`~/.config/QtProject/qtcreator/`）

- **qtversion.xml**：注册系统 Qt5 `/usr/lib/qt5/bin/qmake`（Id=6）
- **profiles.xml**：「PC」Kit 使用 x86_64 GCC + 系统 Qt5（Id=6）

## 五、文字不显示的根因

**问题**：Qt Creator 内置输出面板不是一个真正的 TTY 终端，导致 fontconfig 和 Qt 字体渲染引擎初始化不完整，所有文字渲染为空。

**修复**：`RunConfiguration.UseTerminal = true`（在独立终端窗口中运行）。

> 之前误判为字体缺失/QSS 字体设置等，均已回退。
