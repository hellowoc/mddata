# 环形缓冲区详解

## 它解决什么问题

串口发数据，速度和时序不匹配：

```
GlobalFlow（主线程）:          硬件串口:
  → 一瞬间产生几十条指令         → 一条一条发，每条几毫秒
  → 不能等着串口慢慢发（会卡UI）   → 跟不上 GlobalFlow 的产出速度
```

需要有仓库暂存——GlobalFlow 产出的指令先放仓库，发送线程从仓库慢慢取出来发。

---

## 为什么叫"环形"

普通数组用满了 rear 越界，需要搬数据。环形缓冲区靠取模运算让 rear 到末尾时回到 0：

```
物理内存: [0] [1] [2] ... [4999]  （一段连续内存）
逻辑结构: [0] → [1] → [2] → ... → [4999]
           ↑                       │
           └───────────────────────┘  （首尾相连成环）
```

## 这个项目的实际数据结构

```cpp
#define QUEUE_SIZE 5000        // 5000 个槽位

typedef struct _SqQueue {
    char data[QUEUE_SIZE][SEND_PACKET_SIZE];  // 5000 行，每行一个数据包
    unsigned int front;                        // 队头：下一个要读的位置
    unsigned int rear;                         // 队尾：下一个要写的位置
} SqQueue;
```

---

## front 和 rear 的含义

```
         front=2               rear=5
           ↓                     ↓
  ┌───┬───┬───┬───┬───┬───┬───┬───┐
  │   │   │ C │ D │ E │   │   │   │
  └───┴───┴───┴───┴───┴───┴───┴───┘
           └── 有数据 ──┘

front: 指向第一个有数据的槽（即将被 pop）
rear:  指向第一个空槽（即将被 push）
front == rear → 队列为空
(front - rear) 的区域 = 已写入尚未取走的数据
```

---

## push 操作（入队）

```cpp
bool MySerialQueue::_push(char* data)
{
    // ① 判满：rear 再走一步就到 front → 满了
    if ((m_queue->rear + 1) % QUEUE_SIZE == m_queue->front)
        return FALSE;

    // ② 拷贝数据到 rear 指向的槽位
    memcpy(m_queue->data[m_queue->rear], data, SEND_PACKET_SIZE);

    // ③ rear 向前走一步（取模实现环形）
    m_queue->rear = (m_queue->rear + 1) % QUEUE_SIZE;
    return TRUE;
}
```

```
push 前: [...][D][E][ ][ ][ ]
              ↑front  ↑rear

push(F):  [...][D][E][F][ ][ ]
              ↑front     ↑rear  (rear 前进一格)
```

---

## pop 操作（出队）

```cpp
bool MySerialQueue::_pop(char* data)
{
    // ① 判空：front == rear
    if (m_queue->rear == m_queue->front)
        return FALSE;

    // ② 从 front 槽位拷贝数据出来
    memcpy(data, m_queue->data[m_queue->front], SEND_PACKET_SIZE);

    // ③ front 向前走一步
    m_queue->front = (m_queue->front + 1) % QUEUE_SIZE;
    return TRUE;
}
```

```
pop 前: [...][D][E][F][ ][ ]
              ↑front     ↑rear

pop → 拿到 D: [...][ ][E][F][ ][ ]
                  ↑front ↑rear  (front 前进一格)
```

---

## `% QUEUE_SIZE` — 环形怎么实现

```
rear 在 4999: (4999 + 1) % 5000 = 0    ← 绕回开头
rear 在 4998: (4998 + 1) % 5000 = 4999 ← 正常前进
rear 在 0:    (0 + 1) % 5000 = 1       ← 正常前进
```

---

## 为什么留一个空槽

```cpp
if ((m_queue->rear + 1) % QUEUE_SIZE == m_queue->front)
    return FALSE;  // 满了
```

5000 个槽，最多存 4999 个数据。因为：

```
存满 5000 个: front == rear  ← 和"空"的条件完全一样 → 无法区分

留一个空槽:
  空: front == rear
  满: (rear + 1) % SIZE == front  ← 两条不同的判断，能区分
```

## 完整数据流

```
GlobalFlow                          mySerialSendThread
    │                                    │
    ├─ myQueue.push(packet1) ──→ 队列:    │
    ├─ myQueue.push(packet2)      [1][2]  │
    └─ myQueue.push(packet3)      [1][2][3]
                                     │    │
                                     │    ├─ myQueue.pop() → packet1 → write(串口)
                                     │    ├─ myQueue.pop() → packet2 → write(串口)
                                     │    └─ myQueue.pop() → packet3 → write(串口)

push 和 pop 可能同时发生 → QMutex 保证同一时刻只有一个线程操作 front/rear
```

## 性能对比

| 方式 | push/pop 操作 | 适用场景 |
|------|-------------|---------|
| 普通数组 | O(n) — 每次出队要搬动全部元素 | 不适用 |
| 环形缓冲区 | O(1) — 只改 front/rear 指针 | ✓ 本项目选择 |
| 链表队列 | O(1) — new/delete 节点 | 节点大小不一时更好 |
| QQueue | O(1) — Qt 封装好的队列 | 更方便但引入额外依赖 |

---

## 一句话

环形缓冲区 = 一条首尾相连的传送带。放东西的往 rear 位置放，rear 往前走；取东西的从 front 位置取，front 往前走。两个指针走到末尾就绕回 0，周而复始。QMutex 保证放取互不干扰。
