# Qt 的伙伴机制（Buddy）

## 它是什么

**伙伴（Buddy）** 是 Qt 中一种快捷方式机制：把一个 `QLabel` 和另一个控件（如 `QLineEdit`）绑定在一起，用户按下快捷键时，焦点自动跳到那个控件。

---

## 具体效果

```
界面显示:
  ┌──────────────────────────────────┐
  │  &名称： [________________]      │  ← 标签上的字母 N 带下划线
  │                                    │
  │  &密码： [________________]      │  ← 标签上的字母 P 带下划线
  └──────────────────────────────────┘

用户按 Alt+N → 焦点跳到"名称"输入框
用户按 Alt+P → 焦点跳到"密码"输入框
```

---

## 代码实现

```cpp
QLabel *nameLabel = new QLabel("&名称：");     // & 表示 N 是快捷键
QLineEdit *nameEdit = new QLineEdit();

nameLabel->setBuddy(nameEdit);                // 绑定伙伴
// 现在按 Alt+N → 焦点自动跳到 nameEdit
```

---

## 工作机制

```
1. QLabel 上设置文本 "&名称："
     ↓
   Qt 自动识别 & 后面的字符 N 作为快捷键
   界面上 N 字母显示下划线（表示可触发）

2. 调用 nameLabel->setBuddy(nameEdit)
     ↓
   建立了 QLabel → QLineEdit 的绑定关系

3. 用户按 Alt+N
     ↓
   Qt 找到 N 是 nameLabel 的快捷键
     ↓
   Qt 检查 nameLabel 有没有设置 buddy
     ↓
   有 → 调用 buddy->setFocus() → 焦点跳到 nameEdit
   没有 → 快捷键无效
```

---

## 为什么需要这个机制

在桌面应用中，`QLabel` 本身不接受键盘输入，但用户希望用键盘快速定位到输入框。如果不用伙伴机制，你得自己写 `QShortcut` 或者 `eventFilter` 处理每一个快捷键。伙伴机制一行代码搞定。

---

## 在这个项目中的潜在使用

在工厂设置页面中，有大量参数输入框：

```
屏幕上的效果:
  语言选择  [▼ 中文          ]     ← 标签 + 下拉框
  层数设置  [  2层            ]     ← 标签 + 输入框
  速度设置  [  120            ]     ← 标签 + 输入框
```

虽然嵌入式设备主要靠触摸，但 Qt Designer 拖放的 UI 文件中通常都会设置伙伴关系。

---

## 一句话

伙伴机制 = QLabel 上标记一个快捷键（用 `&`），绑定到另一个控件。用户按快捷键时，焦点跳到被绑定的控件。本质就是用 QLabel 给输入框配了一个键盘快捷入口。
