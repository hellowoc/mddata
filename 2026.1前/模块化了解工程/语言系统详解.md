# 语言系统详解

## 一、整体架构

```
                        语言切换流程

用户触发语言切换
    ↓
g_languageIndex = LANG_ENG    ← 全局变量（存在 QSettings 中）
    ↓
g_myLan().ChangeLanguage()    ← 单例调用
    ↓
├── ① 加载对应 .ini 文件 → m_LanMap (QHash)
├── ② initLangTs() → 逐一把 QHash 值赋给成员变量
├── ③ 写入 QSettings 持久化 g_languageIndex
└── ④ 发送 QEvent::LanguageChange
        ↓
   所有 BaseUi 子类的 changeEvent() 被触发
        ↓
   retranslateUiWidget() 被调用
        ↓
   界面上的所有文字刷新为当前语言
```

---

## 二、语言文件的格式

每个语言对应一个 `.ini` 文件（放在 `./lang/` 目录下）：

```ini
; Chinese.ini
ids_conveyor=传送带
ids_select=精度
ids_feed=产量
ids_valve=气阀
ids_ok=确定
ids_cancel=取消
ids_shutdown=关机
```

```ini
; English.ini
ids_conveyor=Conveyor Belt
ids_select=Precision
ids_feed=Feed Rate
ids_valve=Valve
ids_ok=OK
ids_cancel=Cancel
ids_shutdown=Shutdown
```

同样的 key，不同的 value。key 是内部标识符，value 是对应语言的显示文字。

## 四、loadLan() — 把 .ini 文件加载到 QHash

```cpp
void MyLanguage::loadLan(QString filePath)
{
    m_LanMap.clear();
    QFile lanFile(filePath);                    // 如 "./lang/English.ini"
    QTextStream in;
    in.setDevice(&lanFile);

    while (!in.atEnd()) {
        lineStr = in.readLine();                // "ids_conveyor=Conveyor Belt"
        if (lineStr.contains("=")) {
            lineStrList = lineStr.split("=");   // ["ids_conveyor", "Conveyor Belt"]
            m_LanMap.insert(
                lineStrList[0].simplified(),    // key: "ids_conveyor"
                lineStrList[1].simplified()     // value: "Conveyor Belt"
            );
        }
    }
}
```

加载完成后：

```
m_LanMap["ids_conveyor"]  = "Conveyor Belt"
m_LanMap["ids_select"]    = "Precision"
m_LanMap["ids_feed"]      = "Feed Rate"
```

## 五、initLangTs() — 把 QHash 值赋给成员变量

```cpp
void MyLanguage::initLangTs()
{
    ids_conveyor        = m_LanMap.value("ids_conveyor");
    ids_select          = m_LanMap.value("ids_select");
    ids_feed            = m_LanMap.value("ids_feed");
    ids_ok              = m_LanMap.value("ids_ok");
    msg_communicating   = m_LanMap.value("msg_communicating");
    // ... 几百行赋值 ...
}
```

这样做的目的：让业务代码通过 `g_myLan().ids_conveyor` 成员访问拿翻译文字，而不是在每个地方写 `m_LanMap.value("ids_conveyor")`。成员变量访问有编译期检查，写错了会编译报错；QHash 查找写错了 key 字符串只有运行时才知道。

## 六、ChangeLanguage() — 切换语言

```cpp
void MyLanguage::ChangeLanguage()
{
    // 1. 卸载旧翻译器
    delete translator;
    translator = new QTranslator;

    // 2. 根据 g_languageIndex 选择 .ini 文件
    switch (g_languageIndex) {
    case LANG_CHS:  qmFileName = "./lang/Chinese.ini";    break;
    case LANG_ENG:  qmFileName = "./lang/English.ini";    break;
    // ... 23 种语言 ...
    }

    // 3. 设置字体（泰文/缅甸语等需要特殊字体）
    if (g_languageIndex == LANG_THAILAND)
        font.setFamily("Leelawadee");
    else
        font.setFamily("WenQuanYi Micro Hei");

    // 4. 持久化语言选择
    QSettings setting(CFG_APPSet, QSettings::IniFormat);
    setting.setValue("languageindex", g_languageIndex);
    setting.sync();

    // 5. 重新加载
    loadLan(qmFileName);           // 读 .ini → m_LanMap
    initLangTs();                  // m_LanMap → 成员变量

    // 6. 发送 LanguageChange 事件 → 触发所有页面刷新
    QEvent ev(QEvent::LanguageChange);
    QCoreApplication::sendEvent(qApp, &ev);
}
```

## 七、语言切换后界面如何刷新

```
QCoreApplication::sendEvent(qApp, LanguageChange)
    │
    ▼
每个 BaseUi 子类的 changeEvent() 被触发
    │
    ├── ZKSort::changeEvent
    ├── TopWidget::changeEvent
    ├── BottomWidget::changeEvent
    └── ... 所有页面和控件 ...
           │
           ▼
        retranslateUiWidget()
           │
           ├── ui->label->setText(g_myLan().ids_select)
           ├── ui->btn->setText(g_myLan().ids_ok)
           └── 界面上的文字全部刷新为当前语言
```

## 八、语言偏好如何持久化

```cpp
// 读取（构造函数）
g_languageIndex = setting.value("languageindex", 1).toInt();
//                                       默认值 = 1 = English

// 写入（ChangeLanguage 中）
setting.setValue("languageindex", g_languageIndex);
setting.sync();    // 立刻刷盘，断电不丢
```

## 九、完整时序图

```
开机
  ↓
MyLanguage 构造函数
  → 从 QSettings 读 g_languageIndex（上次用户选择的语言）
  ↓
main.cpp 中:
  paramDelayCode.getUiLanguage();    ← 可选：从 EEPROM 读语言索引
  g_myLan().ChangeLanguage();        ← 加载对应的 .ini 文件
  ↓
所有页面初始化
  → retranslateUiWidget() 中通过 g_myLan().xxx 取文字
  ↓
用户进入语言设置页面 → 选择新语言
  → g_languageIndex = LANG_CHS
  → g_myLan().ChangeLanguage()
  → 界面立即刷新为中文
  → 语言选择写入 QSettings + EEPROM
  ↓
下次开机 → 自动以中文启动
```

## 十、这个方案的特点

| 特点 | 说明 |
|------|------|
| **不用 Qt 标准翻译机制** | Qt 的 `.ts/.qm` 方案没被采用，项目用 `.ini` + 成员变量 |
| **23 种语言** | 从中文到阿拉伯文到泰文，覆盖主要出口市场 |
| **简单直接** | `.ini` 文本文件，非技术人员也能编辑翻译 |
| **编译期检查** | 通过成员变量名访问，写错立即编译报错 |
| **热切换** | 切换语言不需要重启，ChangeLanguage() 后立即刷新 |
| **字体适配** | 不同语言自动切换字体（泰文用 Leelawadee，中文用文泉驿微米黑） |

---

## 一句话

这个语言系统 = 一个装了 23 种语言翻译表的巨型对象（MyLanguage 单例）。翻译表以 `.ini` 文件存储，切换语言时重新加载对应的 `.ini` → 赋给成员变量 → 发送 LanguageChange 事件 → 所有 UI 页面的 `retranslateUiWidget()` 被自动调用 → 界面上的所有文字刷新为新语言。
