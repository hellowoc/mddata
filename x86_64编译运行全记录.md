### 4. 编译主工程 zksort

**遇到的问题和修复：**

#### 问题 1：narrowing conversion 编译错误

Qt5/C++11 下 `char c = fileSize % 256` 触发 `-Wnarrowing` 错误。

修复：在 `zksort/zksort.pro` 中添加编译选项：
```qmake
QMAKE_CXXFLAGS += -Wno-narrowing
```

#### 问题 2：encrypt 加密库是 ARM 版本，无法链接

`3rdparty/encrypt/` 下的 `libtime_encrypt.a` 和 `libmyeeprom.a` 是 ARM aarch64 预编译二进制，没有源码。

修复：创建 x86_64 stub 实现。

**`3rdparty/encrypt/time_encrypt_stub.cpp`**：
```cpp
/* 解密函数 stub — 始终返回 "解密成功"，到期日设为 2099-12-31 */
int DecryptCode(const char* code, unsigned short* wYear,
                unsigned short* wMonth, unsigned short* wDay, char* onlyID)
{
    if (wYear)  *wYear  = 2099;
    if (wMonth) *wMonth = 12;
    if (wDay)   *wDay   = 31;
    if (onlyID) strcpy(onlyID, "X86STUB");
    (void)code;
    return 0; /* decrypt_succeed */
}

bool WhetherCodesMatch(const char* code1, const char* code2)
{
    (void)code1; (void)code2;
    return true;
}
```

**`3rdparty/encrypt/myeeprom_stub.cpp`**：
```cpp
/* EEPROM stub — 用文件 /tmp/eeprom_stub.bin 模拟 I2C EEPROM 硬件 */
// 实现 myeeprom_read / myeeprom_write / writeSysTimeToE2prom / getSysTimeFromE2prom 等
```

编译 stub 库：
```bash
cd /home/jie/桌面/aiuse/3rdparty/encrypt
rm -f libtime_encrypt.a libmyeeprom.a time_encrypt.a
g++ -c -O2 time_encrypt_stub.cpp -o time_encrypt_stub.o
ar rcs libtime_encrypt.a time_encrypt_stub.o
g++ -c -O2 myeeprom_stub.cpp -o myeeprom_stub.o
ar rcs libmyeeprom.a myeeprom_stub.o
```

编译主工程：
```bash
cd /home/jie/桌面/aiuse/zksort
qmake zksort.pro
make -j$(nproc)
```

产物：`zksort/dist/zksort`（ELF 64-bit x86-64, 7MB）

## 运行环境准备

### 1. 创建嵌入式路径

原程序写死了 `/sdcard/` 路径（ARM 嵌入式 Linux 的 SD 卡挂载点）：

```bash
sudo mkdir -p /sdcard/logs /sdcard/cnf /sdcard/ts /sdcard/logo
sudo chmod -R 777 /sdcard
```

### 2. 创建虚拟串口

原程序需要 `/dev/ttySAC1` 和 `/dev/ttySAC2`（Samsung ARM 平台串口）：

```bash
sudo apt-get install -y socat

# 后台启动两个虚拟串口对
sudo socat PTY,link=/dev/ttySAC1,raw,echo=0,mode=666 \
          PTY,link=/dev/ttySAC1_out,raw,echo=0,mode=666 &
sudo socat PTY,link=/dev/ttySAC2,raw,echo=0,mode=666 \
          PTY,link=/dev/ttySAC2_out,raw,echo=0,mode=666 &
```

### 3. 设置库路径并启动

```bash
cd /home/jie/桌面/aiuse/zksort/dist
LD_LIBRARY_PATH=../../3rdparty/log4qt/lib/unix:../../3rdparty/qextserialport/lib/unix:../../3rdparty/jsoncpp/lib/unix:../../uikits/lib/unix \
    ./zksort
```

### 4. 非致命警告（不影响运行）

| 警告 | 原因 | 影响 |
|------|------|------|
| `SIOCSIFFLAGS: 不允许的操作` | 配置网络接口需要 root | 无，网络功能降级 |
| `cannot create /proc/sys/net/core/rmem_default` | 调整网络缓冲区需要 root | 无 |
| `getProfile(): can not get profile parameters` | 无方案配置文件 | 使用默认参数 |

## 后续注意事项

1. **重启后需要重建**：`/dev/ttySAC1` 和 `/dev/ttySAC2` 是 socat 创建的临时符号链接，重启后需要重新启动 socat
2. **加密功能为 stub**：`time_encrypt` 和 `myeeprom` 是文件模拟版本，实际加密校验和 I2C EEPROM 读写不会真正执行
3. **网络功能受限**：部分 UDP socket 和 MQTT 功能可能需要 root 权限才能正常绑定端口
4. **方案配置文件**：启动时提示 `getProfile(): can not get profile parameters, using default`，如需完整功能需要放置 `.cnf` 配置文件到 `/sdcard/cnf/`
