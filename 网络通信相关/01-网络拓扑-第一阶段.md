# 第一阶段：认识色选机的网络拓扑

## 1. 先看物理连接——你的 Linux 工控机有两张网卡

在机器信息界面（`CameraTest(aiuse)/zksort/systeminfo/machineinfowidget.cpp:70-82`），你能看到两个 IP 地址的显示：

```cpp
QSettings setting(CFG_APPSet, QSettings::IniFormat);
setting.beginGroup("eth0");
QString ipStr0 = setting.value("ipStr").toString();  // 第一张网卡的 IP
setting.endGroup();
setting.beginGroup("eth1");
QString ipStr1 = setting.value("ipStr").toString();  // 第二张网卡的 IP
setting.endGroup();

str = "IP1 : " + ipStr0;   // 显示在界面上
str = "IP2 : " + ipStr1;
```

这两张网卡各连接一个独立的物理网络：

```
┌──────────────────────────────────────────────────────────┐
│                    你的 Linux 工控机                       │
│                                                          │
│   eth0: 162.254.129.100  ──── 网线 ──── FPGA 相机板       │
│         子网掩码 255.255.255.0                             │
│         网段范围 162.254.129.1 ~ 162.254.129.254           │
│                                                          │
│   eth1: 192.168.1.100    ──── 网线 ──── IPC AI 工控机      │
│         子网掩码 255.255.255.0                             │
│         网段范围 192.168.1.1 ~ 192.168.1.254               │
└──────────────────────────────────────────────────────────┘
```

### 为什么要两张网卡？

因为两个下游设备需要**不同网段**：

- **FPGA 相机板**用的是 162.254.129.x 网段（厂家自定义的私有网段）
- **IPC AI 工控机**用的是 192.168.1.x 网段（常见的局域网段）

如果它们在同一网段，IP 冲突且数据互相干扰。物理隔离确保相机图像数据的高速传输不受 AI 命令交互的影响。

### 什么是 IP 地址？什么是子网掩码？

可以这样类比：IP 地址就像**房间号**，子网掩码决定了**哪些房间在同一栋楼里**。

- `162.254.129.100` 是房间号
- `255.255.255.0` 表示前三个数字相同的 IP 都在同一栋楼（同一子网）
- 同一栋楼里的设备可以直接通信，不需要经过路由器

### 什么是端口？

IP 地址找到一台设备，**端口**找到这个设备上的具体程序。类比：

```
IP 地址 = 写字楼地址（找到这栋楼）
端口号  = 公司门牌号（找到楼里的具体公司）
```

同一台设备上可以跑多个网络程序，每个绑定不同的端口：

```
162.254.129.100:8081  →  FPGA 相机 UDP 接收线程
162.254.129.100:8082  →  IPC 命令 UDP 接收线程
192.168.1.100:20001   →  IPC 高速抓图 UDP 接收线程
192.168.1.100:20000   →  IPC 快速命令 UDP 接收线程
```

## 2. 三个 UDP 通信通道

代码中定义了 **三个独立的 UDP 通道**，各自处理不同的通信需求。下面逐一对照代码看它们是怎么初始化的。

### 通道 1：FPGA 相机通道（eth0）

**代码位置**：`CameraTest(aiuse)/zksort/bus/mynetwork.cpp:2352-2424`

负责：从 FPGA 相机板接收**波形数据、图像数据、喷阀计数**。

```cpp
void NetWork::udpBind(QString ip, QString port)
{
    // 创建 BSD socket（注意：不是 Qt 的 QUdpSocket，是 Linux 原生 socket）
    sockfd = socket(AF_INET, SOCK_DGRAM, 0);     // SOCK_DGRAM = UDP

    // 绑定到 eth0 的 IP
    addr_local.sin_port = htons(8081);           // 本地端口 8081
    addr_local.sin_addr.s_addr = inet_addr("162.254.129.100");  // 本机 eth0 IP
    bind(sockfd, ...);                           // 绑定——开始监听这个端口

    // 设置发送目标——FPGA 相机板
    inet_pton(AF_INET, "162.254.129.10", &addr_remote.sin_addr); // 远端 = FPGA
}
```

**关键信息**：
| 项目 | 值 |
|------|-----|
| 本机绑定 | `162.254.129.100:8081` |
| 远端目标 | `162.254.129.10`（FPGA 相机板） |
| Socket 类型 | BSD socket（Linux 原生） |
| 接收缓冲 | 8MB（为了不丢图像帧） |

### 通道 2：IPC AI 命令通道（eth0）

**代码位置**：`CameraTest(aiuse)/zksort/bus/myudpthread.cpp:911-917` + `myudpthread.cpp:925-928`

负责：向 IPC AI 工控机发送**配置命令、状态查询、模型加载**等控制指令。

```cpp
// BackgroudObj 构造函数 — 接收 socket
BackgroudObj::BackgroudObj(QObject *parent, int listenPort, SendThread* pAttach)
{
    udpSocket = new QUdpSocket(this);        // 这次是 Qt 的 QUdpSocket
    m_bIsBind = udpSocket->bind(
        QHostAddress("162.254.129.100"),     // 绑定到 eth0
        listenPort                            // 端口 8082
    );
}

// 发送数据时
qint64 BackgroudObj::realSendData(const QByteArray &sendBuf, ...)
{
    strPeerIP = "162.254.129.10";            // 目标也是 162.254.129.10
    return udpSocket->writeDatagram(sendBuf,
        QHostAddress(strPeerIP),
        m_pAttach->remotePort);              // 远端端口 5001
}
```

**关键信息**：
| 项目 | 值 |
|------|-----|
| 本机绑定 | `162.254.129.100:8082`（收）/ `8081`（发） |
| 远端目标 | `162.254.129.10:5001` |
| Socket 类型 | Qt QUdpSocket |
| 协议 | 自定义二进制帧，12 字节头 + 数据 + CRC |

### 通道 3：IPC 高速抓图通道（eth1）

**代码位置**：`CameraTest(aiuse)/zksort/bus/myfastnetwork.cpp:54-77`

负责：从 IPC AI 工控机高速接收**相机图像数据**，用于行连续性测试和全帧抓图。

```cpp
void MyFastNetWorkThread::udpSocketInit(QString ip, QString port)
{
    sockfd = socket(AF_INET, SOCK_DGRAM, 0);       // Linux 原生 socket

    // 接收缓冲区设为 8MB（高速图像传输不能丢包）
    int valueTemp = 8 * 1024 * 1024;
    setsockopt(sockfd, SOL_SOCKET, SO_RCVBUF, &valueTemp, sizeof(valueTemp));

    // 绑定到 eth1 的 IP
    addr_local.sin_port = htons(20001);             // 本地端口 20001
    addr_local.sin_addr.s_addr = inet_addr("192.168.1.100");  // 本机 eth1 IP
    bind(sockfd, ...);
}
```

**关键信息**：
| 项目 | 值 |
|------|-----|
| 本机绑定 | `192.168.1.100:20001`（收） |
| 远端目标 | 动态，每台 IPC 相机一个 IP：`192.168.1.x:20001` |
| Socket 类型 | BSD socket（Linux 原生） |
| 用途 | 行连续性测试 + 全帧图像抓取 |

## 3. 三通道全景图

```
┌──────────────────────────────────────────────────────────────────┐
│                        你的 Linux 工控机                          │
│                                                                  │
│  eth0: 162.254.129.100                                          │
│  ├── :8081 ──→ 通道1(FPGA): 收波形/图像 ←── 162.254.129.10      │
│  ├── :8081 ──→ 通道2(IPC):  发UDP命令  ──→ 162.254.129.10:5001  │
│  └── :8082 ←── 通道2(IPC):  收UDP回包  ←── 162.254.129.10       │
│                                                                  │
│  eth1: 192.168.1.100                                            │
│  ├── :20001 ←─ 通道3(高速): 收相机图像 ←── 192.168.1.x          │
│  └── :20000 ←─ 通道3(高速): 收快速命令 ←── 192.168.1.x          │
│                                                                  │
│  MQTT (任意网卡) ──→ cloud.chinaamd.cn:1884 (云端远程控制)       │
│  HTTP (任意网卡) ──→ cloud.chinaamd.cn:8900 (文件/认证)          │
└──────────────────────────────────────────────────────────────────┘
```

### 为什么 FPGA 相机和 IPC 命令都在 eth0 上？

因为它们都连接到 **162.254.129.10** 这个地址——这实际上是同一块硬件板（FPGA 板上有两个逻辑模块，一个负责相机控制，一个负责与 IPC 通信）。它们共享同一根网线但用不同端口区分服务。

### 为什么高速抓图要单独用 eth1？

因为 IPC AI 工控机产生的是**原始相机图像数据**，每帧可能几 MB，每秒几十帧。如果和命令通道共享 eth0，图像流量会淹没命令包，导致控制指令延迟。所以专门用 eth1（192.168.1.x）承载高速图像流。

## 4. 为什么用 UDP 而不是 TCP？

这个工程中**所有对下位机的通信都是 UDP**，原因：

| 对比维度 | TCP | UDP |
|---------|-----|-----|
| 连接 | 需要三次握手建立连接 | 无连接，直接发 |
| 可靠性 | 保证送达、保证顺序 | 不保证送达、不保证顺序 |
| 延迟 | 高（确认重传机制） | 低（发了就忘） |
| 适用场景 | 文件传输、网页浏览 | 实时视频、游戏、工业控制 |

色选机场景：
- **实时性优先**：物料快速流过相机，丢一帧无所谓，下一帧马上到
- **协议简单**：不需要 TCP 的流控和拥塞控制，自定义二进制协议够用
- **CRC 自校验**：自定义协议在应用层做了 CRC，不依赖 TCP 的校验

## 第一阶段的三个核心理解

1. **两张网卡物理隔离**：eth0 接 FPGA 相机（162.254.129.x），eth1 接 IPC AI 工控机（192.168.1.x）
2. **三个 UDP 通道各司其职**：命令通道（端口 5001）、图像通道（端口 20001）、波形通道（端口 8081）
3. **全 UDP 是因为工业场景要低延迟**，丢一两个包可以接受，协议自带 CRC 校验
