## 1. 这个通道是干什么的

IPC 高速抓图通道负责从 **IPC AI 工控机** 高速接收**相机原始图像数据**。它有两个工作模式：

| 模式 | 宏 | 说明 |
|------|----|------|
| 行连续性测试 | `FAST_CAPTURE_IMAGE`(1) | 逐包检测行号是否连续，报告丢行/错包 |
| 全帧抓图 | `FAST_FR_CAPTURE_IMAGE`(2) | 接收完整图像帧，拼成 BMP 存盘 |

### 与通道1、通道2的关系

```
┌─────────────────────────────────────────────────────────────┐
│                      你的 Linux 工控机                        │
│                                                             │
│  eth0 (162.254.129.100)         eth1 (192.168.1.100)        │
│  ├── 通道1: FPGA 波形/图像       ├── 通道3: IPC 高速图像 ←── 本通道 │
│  │    (mynetwork.cpp)           │    (myfastnetwork.cpp)     │
│  └── 通道2: IPC 控制命令         └── :20000 FastNetObj(命令)  │
│       (myudpthread.cpp)                                     │
└─────────────────────────────────────────────────────────────┘
```

**通道2 vs 通道3**：通道2 发**控制命令**（"加载模型"、"开始分选"），通道3 接收**图像数据**（IPC 推理过程中产生的原始图像）。通道2 的命令有一部分是通过通道3 的 FastNetObj（`192.168.1.100:20000`）发出的，但通道3 的 `MyFastNetWorkThread` 是纯接收线程。

## 3. 包头格式

IPC 高速抓图通道的包头与通道2 类似——以 `A5 5A` 起头，但结构不同：

```
偏移  长度  含义
─────────────────────────
0     2     A5 5A 帧头
2     4     图像序号（imageSeq，用于全帧拼图）
6     2     行号（高字节 + 低字节）
8     2     分包序号（一行可能分多个包传输）
10    2     总包数（这一行共几个包）
12    2     当前像素数（这个包中有多少像素）
14    ...   像素数据（RGB 或灰度，取决于相机配置）
```

**定长部分**：`PACKET_HEAD_LEN = 14` 字节。

### Step 3：udpSocketInit() — 创建 BSD socket + 绑定

`myfastnetwork.cpp:54-77`：
```cpp
void MyFastNetWorkThread::udpSocketInit(QString ip, QString port)
{
    // 1. 创建 BSD socket
    sockfd = socket(AF_INET, SOCK_DGRAM, 0);

    // 2. SO_REUSEADDR
    int val = 1;
    setsockopt(sockfd, SOL_SOCKET, SO_REUSEADDR, &val, sizeof(val));

    // 3. SO_RCVBUF — 8MB 接收缓冲区
    val = 8 * 1024 * 1024;
    setsockopt(sockfd, SOL_SOCKET, SO_RCVBUF, &val, sizeof(val));

    // 4. 填充 sockaddr_in + bind
    memset(&addr_local, 0, sizeof(addr_local));
    addr_local.sin_family = AF_INET;
    addr_local.sin_port = htons(20001);                  // 端口 20001
    addr_local.sin_addr.s_addr = inet_addr("192.168.1.100");  // eth1 IP
    bind(sockfd, (struct sockaddr*)&addr_local, sizeof(addr_local));
}
```

### Step 4：udpServerInit() — 初始化远端地址表

`myfastnetwork.cpp:79-102`：
```cpp
void MyFastNetWorkThread::udpServerInit(int ipIndex)
{
    // 遍历所有层/视野/单元，为每台 IPC 相机建立远端地址
    for (i = 0; i < nLayerTotal; i++)
        for (j = 0; j < nViewChainTotal; j++)
            if (nViewIsUsed[j] == 1)
                for (k = 0; k < nViewUnitTotal[j]; k++)
                    if (ipcType != IPC_CAMERA_TYPE_NULL)
                    {
                        // 从配置中读取 IPC 的 IP 地址：192.168.1.x
                        QString serverIp = QString("192.168.1.%1")
                            .arg(struCnfg.struIpcCameraParams[i][j][k].nIpAddr[4]);

                        // 填充 addr_remote[j][k]
                        addr_remote[j][k].sin_family = AF_INET;
                        inet_pton(AF_INET, ip, &addr_remote[j][k].sin_addr);
                        addr_remote[j][k].sin_port = htons(20001);
                    }
}
```

**远端地址是二维数组** `addr_remote[view][unit]`——每台 IPC 相机有独立的 IP 地址（`192.168.1.10`、`192.168.1.11`...），通过 `struCnfg.struIpcCameraParams` 配置。

## 5. 两种工作模式

### 模式 1：行连续性测试（FAST_CAPTURE_IMAGE = 1）

**目的**：检测 IPC 相机传输过程中是否丢行。

**触发方式**：IPC 信息页（`ipcinfowidget.cpp`）点击"行连续性测试"按钮。

**流程**：
```
1. ipcinfowidget 调用 g_Udp.pushUdpCmdData(CMD_UDP_HT_GET_FAST_CAPTURE, ...)
   通过 FastNetObj → 192.168.1.x:20000 发命令 → IPC 开始推送图像数据

2. myFastNetThread->isrunning = true;
   myFastNetThread->setCurrentMode(FAST_CAPTURE_IMAGE);
   myFastNetThread->start();
```

**run() 中的接收逻辑**：
```cpp
while (isrunning)
{
    // select() 检查数据
    checkUdpSocketStatues(seconds);
    
    // recvfrom() 收包
    retNum = recvfrom(sockfd, &packetBuf[curTotal], ...);
    curTotal += retNum;

    // 检查是否有完整包（至少 PACKET_HEAD_LEN 字节）
    while (curTotal >= PACKET_HEAD_LEN)
    {
        // 校验帧头
        if (packetBuf[0] != 0xA5 || packetBuf[1] != 0x5A)
        {
            丢弃前部字节，重新对齐;
            continue;
        }

        // 提取行号（字节18-21，大端序）
        rowNum = packetBuf[18]<<24 | packetBuf[19]<<16 
               | packetBuf[20]<<8  | packetBuf[21];

        // 与上一包的行号比较
        expectedRow = lastRowNum + 1;
        if (rowNum != expectedRow)
        {
            lostLines++;    // 丢行了！
            headerErrors++; // 包头错
        }
        lastRowNum = rowNum;
    }
}
// 测试结束，emit captureTestResult(view, unit, lostLines, totalLines);
// → ipcinfowidget::handleCaptureTestResult() 更新界面显示
```

### 模式 2：全帧抓图（FAST_FR_CAPTURE_IMAGE = 2）

**目的**：从 IPC 相机接收完整的一帧图像，保存为 BMP 文件。

**流程**：
```
1. ipcinfowidget 调用 myFastNetThread->setCurrentMode(FAST_FR_CAPTURE_IMAGE)
2. 调用 start() 启动线程
3. run() 中：
   - 每个 UDP 包携带一行的一部分像素
   - 一行图像需要 m_packetPerLine 个包（3 或 6，取决于像素数）
   - 按 imageSeq（包内序号）拼帧
```

**run() 中的接收逻辑**：
```cpp
// 每收到一个包
imageSeq = 从包头字节 2-5 提取;    // 图像序号（哪一帧）
packetIndex = 从包头提取;          // 本行中的第几个分片

// 在 imageCombineBuf 中按 imageSeq 和 packetIndex 定位写入位置
// 一行 1024 像素 × 3 通道 = 3072 字节
// 每包携带约 1024 字节像素数据
// 所以每行需要 3072/1024 ≈ 3 个包

// 全部收完后
saveBMPFile(path, width, height);   // 拼成 BMP 存到 /sdcard/bmp/

// 发射信号通知 UI
emit SignalImageRecvFinished(filePathList, timeCostList);
```

## 6. 与通道2 的协作

通道3 本身**只负责接收图像数据**，不发命令。启动和停止由通道2 的 `FastNetObj` 负责：

```
IPC 信息页（ipcinfowidget.cpp）
    │
    ├── 启动测试
    │   ├── g_Udp.pushUdpCmdData(CMD_UDP_HT_GET_FAST_CAPTURE, ...)
    │   │   → SendThread::addPacket() 路由到 m_MySocket2 (FastNetObj)
    │   │   → realSendData() → writeDatagram() → 192.168.1.x:20000
    │   │   → IPC 收到命令，开始推送图像
    │   │
    │   └── myFastNetThread->start() → run() 开始收图
    │
    └── 停止测试
        ├── g_Udp.pushUdpCmdData(CMD_UDP_HT_GET_FAST_CAPTURE, stop, ...)
        └── myFastNetThread->isrunning = false
```

## 7. 与通道1 的对比

| | 通道1 (FPGA) | 通道3 (IPC 高速) |
|------|-----------|--------------|
| 网卡 | eth0 | eth1 |
| 本机端口 | 8081 | 20001 |
| 远端 | 162.254.129.10 | 192.168.1.x（多台） |
| 包头 | `FF 00 A5 5A C3 3C`（6字节） | `A5 5A + 12字节头`（14字节） |
| 数据内容 | 波形 + 图像 + 喷阀计数 | 仅图像数据 |
| 工作模式 | 10 种采图模式 | 2 种（行连续性 / 全帧抓图） |
| 远端地址 | 硬编码一个 IP | 每台 IPC 一个 IP，从配置读取 |
| Socket | BSD socket（相同） | BSD socket（相同） |
| 创建时机 | 程序启动 | 程序启动（但不启动线程） |
| 启动时机 | 界面点击采图按钮 | 界面点击测试按钮 |

## 8. 线程跨越点

| 环节 | 所在线程 | 说明 |
|------|---------|------|
| udpSocketInit / udpServerInit | 主线程 | 构造函数中调用 |
| g_Udp.pushUdpCmdData (启动命令) | 主线程 | 通过 FastNetObj 发送 |
| select / recvfrom | 子线程 | `run()` 中死循环 |
| 图像数据处理 | 子线程 | 包头解析、丢行检测、BMP 拼帧 |
| emit captureTestResult | 子线程 | Qt 跨线程调度 |
| saveBMPFile | 子线程 | 文件 I/O，不阻塞 UI |
| 界面更新（槽函数） | 主线程 | 信号触发后执行 |
