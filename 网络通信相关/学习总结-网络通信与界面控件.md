## 知识体系总览

### 一、网络基础

#### 核心概念

| 概念 | 一句话理解 | 详细文档 |
|------|-----------|---------|
| 网卡 (eth0/eth1) | 电脑的"嘴巴和耳朵"，物理硬件 | `00-网络基础概念` |
| MAC 地址 | 出厂烧录，全球唯一（身份证号） | `00b-IP地址与MAC地址` |
| IP 地址 | 软件配置，可以随时改（门牌号） | `00b-IP地址与MAC地址` |
| 子网掩码 | 划分"网络号"和"主机号" | `00-网络基础概念` |
| 同网段通信 | 前三段相同 → 直连，不需要路由器 | `00-网络基础概念` |
| 端口 | IP 找设备，端口找程序（房间号） | `00-网络基础概念` |
| UDP | 无连接、不保证送达、延迟低（对讲机） | `00-网络基础概念` |
| TCP | 三次握手、保证送达、延迟高（打电话） | `00-网络基础概念` |
| ping | 验证 IP 层连通性 | `10-远程控制-基础知识` |
| VPN | 在公网上建加密隧道，虚拟一张网卡 | `10-远程控制-基础知识` |
| DHCP | 自动获取 IP，工业设备一般不用 | `00b-IP地址与MAC地址` |

#### 通信协议栈

```
应用层    → 你的程序
传输层    → UDP / TCP（协议选择）
网络层    → IP（寻址和路由，ping 在这一层）
链路层    → 网卡驱动
物理层    → 网线/电信号
```

### 二、Socket 编程

#### BSD Socket（C 原生 API）

| 函数 | 作用 | 详细文档 |
|------|------|---------|
| `socket()` | 创建 socket，返回文件描述符 | `references/Socket编程基础` |
| `bind()` | 绑定本机 IP+端口 | `references/Socket编程基础` |
| `select()` | 检查 socket 是否可读（非阻塞轮询） | `references/Socket编程基础` |
| `recvfrom()` | 阻塞接收 UDP 包 | `references/Socket编程基础` |
| `sendto()` | 发送 UDP 包 | `references/Socket编程基础` |
| `setsockopt()` | 设置 socket 选项（重用地址、增大缓冲） | `references/Socket编程基础` |
| `htons()` | 字节序转换（主机→网络） | `references/Socket编程基础` |
| `inet_addr()` | 字符串 IP → 32 位二进制 | `references/Socket编程基础` |
| `inet_pton()` | 字符串 IP → 结构体（新版 API） | `references/Socket编程基础` |
| `sockaddr_in` | "地址+端口"结构体 | `references/Socket编程基础` |

**典型流程**：`socket() → setsockopt() → bind() → select() → recvfrom()`

或发送：`sendto()`

| | BSD Socket | Qt QUdpSocket |

### 四、Qt 基础

#### .ui 文件与自动生成代码

| 概念 | 说明 |
|------|------|
| `.ui` 文件 | XML 文件，Qt Designer 拖出来的界面布局 |
| `uic` 编译器 | 把 `.ui` → `ui_xxx.h`（自动生成，不要手动改） |
| `setupUi(this)` | 生成 `new QPushButton`、`new QLabel`、`addWidget` 等代码 |
| `ui->xxx` | `.ui` 中 `name="xxx"` 的控件，自动变成 `ui->xxx` 指针 |
| `namespace Ui { class xxx; }` | 前向声明，避免 `.h` 包含自动生成的大文件 |

| 控件类型 | 常用方法 |

#### 信号与槽

| 机制 | 说明 |
|------|------|
| 手动 `connect` | `connect(发送者, SIGNAL(信号), 接收者, SLOT(槽))` |

#### 核心 Qt 类

| 类 | 作用 |
|------|------|
| QByteArray | 字节数组，代替 `char[]`，自动管理内存 |
| QMap<Key, Value> | 键值对容器，`insert()`/`value()`/`key()` |
| QMutexLocker | 互斥锁 RAII 包装，构造时加锁、离开作用域自动解锁 |
| QThread | 线程基类，重写 `run()`，用 `start()` 启动 |
| QTimer | 定时器，`start(ms)` 循环触發，`setSingleShot(true)` 单次触发 |

| QStackedWidget | 卡片堆，`setCurrentIndex()` 切换显示哪个子 widget |

#### QThread 两种模式

| | 模式一：死循环 | 模式二：事件循环 |
|------|-------------|-----------------|
| 核心函数 | `while(isrunning)` | `exec()` |
| 退出方式 | `isrunning = false` | `quit()` + `wait()` |
| 适合场景 | 持续高速收数据 | 偶尔有命令需要收发 |
| 本工程例子 | MyNetWorkThread、MyFastNetWorkThread | SendThread |

### 十一、学习方法总结

#### 有效的方法

1. **逐层追问**——从一个函数追到 Qt 类，再追到系统调用，直到无法再展开
2. **用 STM32 类比**——`while(1)` 主循环 ↔ QEventLoop、`emit` ↔ ISR 中断
3. **自己复述总结**——"先找网卡，再找 IP"、"IP 找设备，端口找程序"
4. **对照代码学概念**——不是先看理论，而是先看代码里的实际用法

#### 改进方向

1. **先通读再拆解**——看完整个函数，一句话概括它做什么，再回头拆不懂的细节
2. **先猜后验证**——看到 `getSetting()`，先猜"获取配置"，再读代码验证
3. **同类概念不重复溯源**——`QList<QNetworkAddressEntry>` 和 `QList<IPCVersionInfo>` 本质一样
4. **先骨架后血肉**——主线流程搞清楚再追细节，避免拆到一半忘了主线
